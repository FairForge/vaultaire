EU Data Act Compliance

The EU Data Act (Regulation 2023/2854) establishes new rules for cloud service providers regarding data portability, switching, and interoperability. stored.ge is fully compliant with the Data Act's requirements for cloud services, which apply from September 12, 2025.

Article 23 — Switching Between Providers

The Data Act requires cloud providers to support customers in switching to another provider or to on-premises infrastructure. stored.ge makes switching effortless:

  • S3-compatible API: stored.ge implements the industry-standard S3 protocol. Migrating to or from stored.ge requires zero code changes — the same tools, SDKs, and scripts work across all S3-compatible providers.
  • Standard tools work: aws-cli, boto3, rclone, s3cmd, Cyberduck, and any S3-compatible client work with stored.ge without modification. Switch providers by changing an endpoint URL.
  • No proprietary dependencies: We do not require proprietary SDKs, agents, or client software.

Article 24 — No Switching Fees

The Data Act prohibits cloud providers from charging fees that discourage switching. stored.ge goes beyond the minimum requirements:

  • No egress fees for switching: stored.ge does not charge egress fees. Download your data at any time at no additional cost.
  • No data export fees: Exporting your data via the S3 API or dashboard is free.
  • No early termination penalties: Cancel your account at any time with no penalties or additional charges.
  • No hidden migration costs: There are no charges for API calls, data listing, or metadata access during a migration.

Article 25 — Interoperability

The Data Act requires cloud services to ensure interoperability with other services using open interfaces and standards. stored.ge is built on open standards:

  • Full S3 API compatibility: GET, PUT, DELETE, LIST, HEAD, multipart upload, object versioning, pre-signed URLs, and more.
  • Standard authentication: AWS Signature Version 4 (SigV4), the industry-standard authentication mechanism for S3-compatible services.
  • Standard data formats: Objects are stored and returned as-is. No proprietary encoding, wrapping, or transformation.
  • Standard metadata: Object metadata uses standard HTTP headers and user-defined x-amz-meta-* headers, fully portable across providers.

Article 26 — Porting Assistance

The Data Act requires providers to support customers during the porting process with a reasonable transition period. stored.ge provides:

  • 30-day retention after cancellation: After account deletion is requested, all data is retained for 30 days. During this period, you can access and export your data normally or cancel the deletion entirely.
  • Full API access during porting: There is no degradation of service, rate limiting, or restricted functionality during the transition period.
  • Multiple export methods: Export via the S3 API (recommended for large datasets) or download a JSON summary of all account data from the dashboard settings page.
  • Documentation: Our API documentation and S3 compatibility guide are publicly available to assist with migration planning.

Article 27 — No Vendor Lock-in

The Data Act requires cloud providers to remove barriers to switching by using open standards. stored.ge is architecturally lock-in free:

  • Open protocol: The S3 API is an open, widely-implemented protocol supported by hundreds of tools and libraries.
  • No proprietary SDKs: Any S3-compatible SDK or CLI works with stored.ge. We do not develop or require proprietary client software.
  • No proprietary data formats: Objects are stored byte-for-byte as uploaded. No proprietary headers, checksums, or container formats are added.
  • No proprietary features: All stored.ge features use standard S3 operations. Your workflows are portable to any S3-compatible provider.
